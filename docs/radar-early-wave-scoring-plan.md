# Radar 早期波段重估評分重構規劃

## 目標

重構 `stock_ai_bot` 目前 Radar 評分機制，讓系統更早辨識具備 1～2 個月波段上漲與市場重估潛力的股票。

本規劃不新增獨立的 `wave_score` 或 `wave_bonus` 作為外掛分數，而是將「早期上漲卡位」所需的技術、營收、財報、籌碼、題材與族群訊號，內化到現有 Radar `total_score`。

新版總分仍為 100 分：

| 面向 | 權重 |
|---|---:|
| 技術 | 30 |
| 營收 | 20 |
| 財報 | 15 |
| 籌碼 | 15 |
| 題材/新聞 | 15 |
| 族群 | 5 |

核心判斷邏輯：

```text
技術先觸發，營收提供成長證據，財報確認體質改善，
籌碼確認資金進場，題材讓市場願意重估，族群提供擴散。
```

## 現有 Radar 評分問題

目前 Radar 大致使用：

```text
技術 technical：最高 40
營收 revenue：最高 15
籌碼 chip：最高 15
題材 theme：最高 20
族群 market：最高 10
```

目前評分較粗：

- 技術主要看 A/B/C/D 策略命中與 `technical_setup_score`，但較少細分「低檔站回均線」、「洗盤後收復」、「突破後是否過熱」。
- 營收主要看單月 YoY，沒有完整看近 3～5 個月趨勢、YoY 加速、累計營收與營收品質。
- 財報尚未成為 Radar 正式分項，EPS 虧轉盈、虧損收斂、毛利率與營益率改善沒有被充分獎勵。
- 籌碼有法人/大戶評級，但尚未完整整合外資、投信、自營商、TDCC、融資券與洗盤後回補。
- 題材分偏向新聞/來源數量，尚未完整判斷「舊產業被市場重新命名」與基本面交叉驗證。
- 族群分目前偏同產業候選數，尚未納入同題材、指標股、補漲擴散與族群強度。

新版評分要把原本粗分項拆細，不再另外外掛波段分數。

## 實作總覽

### 保留既有入口

保留目前入口與流程：

- `/radar`
- `/radar_more`
- 每日 21:30 Radar 推播
- `/radar --source technical`
- `/radar --source curated`
- `/radar --source financial`
- `/radar --source chip`
- `/scan`

不更動候選來源的主要邏輯，先重構「候選出來後怎麼評分」。

### 新增 Feature Snapshot

在 Radar 評分前，為每檔候選建立 `radar_feature_snapshot`：

```json
{
  "technical": {},
  "revenue": {},
  "financial": {},
  "chip": {},
  "theme_news": {},
  "sector": {}
}
```

資料來源可使用既有：

- 技術面選股策略訊號
- `.cache/technical_daily`
- `.cache/research_structured`
- `.cache/monthly_revenue`
- `.cache/gross_margin.json`
- `.cache/price_metrics.json`
- `.cache/tdcc`
- `.cache/chip_daily`
- 本地新聞資料庫
- company knowledge / theme map / supply chain nodes
- Radar evidence pack

### 新評分函式

將目前 `_score_candidates()` 拆成細項：

```python
_score_technical_detail(item, snapshot) -> ScoreDetail  # 0~30
_score_revenue_detail(item, snapshot) -> ScoreDetail    # 0~20
_score_financial_detail(item, snapshot) -> ScoreDetail  # 0~15
_score_chip_detail(item, snapshot) -> ScoreDetail       # 0~15
_score_theme_news_detail(item, snapshot) -> ScoreDetail # 0~15
_score_sector_detail(item, snapshot) -> ScoreDetail     # 0~5
```

每個評分函式回傳：

```json
{
  "score": 18,
  "reasons": ["站回 MA20", "量比 > 2"],
  "risks": ["短線乖離偏高"],
  "details": {}
}
```

`RadarCandidate` 或輸出 dict 應保存：

```json
{
  "score_components": {
    "technical": 24,
    "revenue": 18,
    "financial": 13,
    "chip": 9,
    "theme": 10,
    "sector": 2
  },
  "score_details": {
    "technical": {},
    "revenue": {},
    "financial": {},
    "chip": {},
    "theme": {},
    "sector": {}
  },
  "key_reasons": [],
  "risk_flags": [],
  "score_caps_applied": []
}
```

## 技術 30 分

技術面重點是「早期轉強」，不是追已經過熱的股票。

### 趨勢轉強 8 分，上限 8

| 條件 | 分數 |
|---|---:|
| 收盤站回 MA5 | +1 |
| 收盤站回 MA10 / MA13 | +1 |
| 收盤站回 MA20 / MA21 | +2 |
| 收盤站回 MA60 | +2 |
| 收盤站回 MA105 / 半年線 | +2 |
| MA5 上彎 | +1 |
| MA10 / MA13 上彎 | +1 |
| MA20 / MA21 走平轉上 | +1 |

### 低檔轉折 5 分，上限 5

| 條件 | 分數 |
|---|---:|
| 距離 60 日低點小於 20%，且剛轉強 | +1 |
| 距離 120 日低點小於 30%，且剛轉強 | +1 |
| 近 20 日完成底部盤整 | +1 |
| 低檔出現 MACD 背離 | +1.5 |
| 低檔 KD 背離或 KD 黃金交叉 | +1 |
| DIF 由負轉正或接近零軸 | +1 |
| 低檔紅 K 突破整理區 | +1.5 |

### 放量確認 5 分，上限 5

| 條件 | 分數 |
|---|---:|
| 量比 > 1.5 | +1 |
| 量比 > 2 | +2 |
| 量比 > 3 | +3 |
| 成交量突破 20 日均量 | +1 |
| 成交量突破 60 日均量 | +1 |
| 價漲量增 | +1 |
| 量增但未爆大量過熱 | +1 |
| 連續 2～3 日溫和放量 | +1 |

### 洗盤後收復 5 分，上限 5

| 條件 | 分數 |
|---|---:|
| 跌破前一個轉折低點後，1～3 日內收復 | +2 |
| 長下影後收回 MA20 / MA21 | +1.5 |
| 跌破 MA20 / MA21 後快速站回 | +1.5 |
| 跌破 MA60 後快速站回 | +2 |
| 急跌後隔日紅 K 反包 | +1.5 |
| 洗盤日放量，收復日量縮或健康放量 | +1 |
| 洗盤後未再破低 | +1 |
| 回測前波平台後轉強 | +1 |

### 突破與延續 4 分，上限 4

| 條件 | 分數 |
|---|---:|
| 突破近 20 日高點 | +1 |
| 突破近 60 日高點 | +1.5 |
| 突破前波大量高點 | +1 |
| 突破箱型上緣 | +1 |
| 回測突破點不破 | +1 |
| 紅 K 實體突破 | +1 |
| 非一日行情，隔日仍守住 | +1 |

### 策略交叉 3 分，上限 3

A/B/C/D 四種策略不要求全部同時命中，應看不同型態訊號是否交叉確認。

| 條件 | 分數 |
|---|---:|
| 命中任一 A/B/C/D 策略 | +1 |
| 命中兩種不同策略族群 | +1 |
| 趨勢策略 + 低檔反轉策略同時出現 | +1 |
| 回測突破 + 急跌收復同時出現 | +1 |
| 均線收復 + MACD/KD 同步轉強 | +1 |

### 技術扣分

| 條件 | 扣分 |
|---|---:|
| 連續 3 根以上漲停後才出現 | -3 |
| 乖離 MA20 超過 25% | -2 |
| 爆量長上影 | -2 |
| 跌破啟動 K 低點 | -3 |
| 放量不漲 | -2 |
| 技術強但基本面完全無支撐 | 技術分最高 20 |

## 總分封頂規則

封頂規則比單純加分更重要，用來避免純技術、純題材或過熱股被推太前。

| 條件 | 封頂 |
|---|---:|
| 技術未觸發 | 總分最高 60 |
| 只有技術，營收/財報弱 | 總分最高 65 |
| 只有題材，營收/財報弱 | 總分最高 60 |
| 題材無法驗證 | 總分最高 70 |
| 籌碼散戶化嚴重 | 總分最高 70 |
| 財報品質差、現金流惡化 | 總分最高 75 |
| 技術已過熱 | 總分最高 75 |
| 短線已連續大漲且無基本面支撐 | 總分最高 70 |

## 分級

| 總分 | 等級 | 說明 |
|---:|---|---|
| 80～100 | A+ | 早期波段高優先，應提高 Top10 機會 |
| 70～79 | A | 波段候選，值得追蹤 |
| 60～69 | B | 有早期訊號，但需要確認 |
| 50～59 | C | 可能只是短線反彈 |
| < 50 | D | 不優先 |

## 標籤規則

符合條件時可加上顯示標籤，不作為獨立分數：

| 標籤 | 判斷概念 |
|---|---|
| 早期轉強 | 技術剛觸發，尚未過熱 |
| 轉機波段 | 營收/財報改善明顯，市場可能重估 |
| 題材重估 | 舊產業被重新命名且有基本面交叉驗證 |
| 籌碼轉強 | 法人/大戶/TDCC 在觸發附近轉強 |
| 僅技術反彈 | 技術有訊號，但營收/財報/題材不足 |
| 題材未驗證 | 有題材但缺少營收、財報、公告或產品證據 |
| 短線過熱 | 技術強但追價風險高 |

## TG 顯示規劃

### Radar 主訊息

目前 Radar 格式：

```text
1. 3002 歐格｜69分｜策略 B/D｜AI 中
   技術 24｜營收 15｜籌碼 0｜題材 20｜族群 10
   電腦及週邊設備業｜候選來源
   技術訊號：...
   AI短評：...
   風險：...
   觀察：...
```

新版建議：

```text
1. 3550 聯穎｜76分｜策略 D｜題材重估
   技術 24｜營收 18｜財報 13｜籌碼 9｜題材 10｜族群 2
   電子零組件業｜候選來源
   技術訊號：D 強勢股急跌收復：D4 急跌或長下影後收復均線
   關鍵線索：站回 MA20/MA60、營收連續轉強、Q1轉盈、外資轉買、AI高速傳輸
   風險：題材占比未驗證、短線乖離偏高
   AI短評：...
   觀察：...
```

本次同步調整顯示與 AI 補強數量：

- `/radar` 主訊息預設顯示 Top 15。
- 每日 21:30 Radar 推播同樣顯示 Top 15。
- 外部來源補強 / AI 短評預設 `ai_top=15`。
- AI 短評保留既有分段機制，每批 5 檔，因此 Top 15 會分成 3 批。
- 若單批 prompt 過大，保留 `normal -> tight -> minimal` 的單檔降級機制。

### Radar 前段摘要


## `/scan` 顯示規劃

`/scan` 不重排原分類，只加標籤與摘要。

原分類中：

```text
2241 艾姆勒 (50.6)
```

可改成：

```text
2241 艾姆勒 (50.6)｜轉機波段
```

報告後段追加：

```text
📈 早期波段候選
2241 艾姆勒｜轉機波段｜技術轉強 + 營收高年增 + 虧損收斂 + EV散熱
3550 聯穎｜題材重估｜站回均線 + 營收連續轉強 + Q1轉盈 + AI高速傳輸
```

## Artifact 保存

`radar_candidates.json` 中每檔候選需保存：

```json
{
  "score_components": {
    "technical": 24,
    "revenue": 18,
    "financial": 13,
    "chip": 9,
    "theme": 10,
    "sector": 2
  },
  "score_details": {
    "technical": {
      "score": 24,
      "reasons": [],
      "risks": [],
      "details": {}
    }
  },
  "key_reasons": [],
  "risk_flags": [],
  "score_caps_applied": [],
  "radar_feature_snapshot": {}
}
```

舊版快取讀取 `_record_to_result()` 必須能 fallback：

- 若沒有 `financial` 分項，預設 0。
- 若沒有 `score_details`，預設空 dict。
- 若舊版 `theme`/`market` 命名不同，保持相容。
