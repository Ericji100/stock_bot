## 一、報告定位

本報告是「價值重估掃描報告」，不是一般個股資訊彙總、不是完整深度研究、不是交易計畫，也不是買賣建議。

本報告的核心任務是判斷：

1. 候選股票中，哪些標的可能正在發生市場標籤重估。
2. 每檔候選股的舊市場標籤與新市場標籤是什麼。
3. 新標籤是否有產品、客戶、營收、財報、法說會、公司公告、正式新聞、法人籌碼或技術面支撐。
4. 候選股是「已驗證重估」、「推論型重估」、「題材觀察」，還是「蹭題材風險偏高」。
5. 哪些股票值得後續執行 `/research 股票`、`/research 股票 --deep` 或單股 `/value_scan 股票`。

價值重估的定義：

一家公司原本被市場用舊產業、舊產品、舊獲利模式或舊估值邏輯看待，但因為新產品、新客戶、新應用、新產業趨勢、營收結構改變、毛利率改善、法人資金重新認同、技術面轉強或題材擴散，使市場開始用更高估值的新標籤重新評價該公司。

請特別注意：

1. 價值重估不等於新聞熱門。
2. 價值重估不等於股價短線上漲。
3. 價值重估不等於同產業沾邊。
4. 價值重估必須能說明「舊標籤 → 新標籤」的轉換邏輯。
5. 若只有題材想像、論壇討論或短線情緒，但沒有產品、營收、財報、公告、法說會、法人籌碼或技術面支撐，不得給高重估分。
6. 若資料不足，必須明確標示「目前資料不足」或「仍待驗證」，不得自行補故事。

## 六、本地量化底稿與 AI 最終判斷規則

候選資料中可能包含：

- `rerating_score`
- `verification_score`
- `local_rerating_composite_score`
- `local_scoring`
- `technical_score`
- `radar_score`
- `theme_score`
- `revenue_score`
- `chip_score`
- `technical_signal_type`
- `sub_signal_type`

以上資料都是本地量化底稿，不是 AI 最終重估結論。

請遵守以下規則：

1. 本地分數只能作為參考底稿，不得直接照抄為 AI 最終分數。
2. AI 必須根據公告、法說會、財報、月營收、產品、客戶、供應鏈、新聞來源、法人籌碼、技術面、反證與資料完整性，重新產生 AI 最終重估排序。
3. 若 AI 排名與本地排序不同，必須說明差異原因。
4. 若 AI 最終重估分明顯高於本地底稿，必須說明新增了哪些證據、哪些是推論型加分、哪些仍待驗證。
5. 若 AI 最終重估分明顯低於本地底稿，必須說明哪些本地分數可能過度樂觀、哪些資料缺口或反證壓低評分。
6. 若資料不足，不得用本地分數、題材想像、低股價、新聞熱度、論壇討論或市場情緒補高 AI 最終分數。
7. 財務硬指標、營收驗證、獲利驗證不得靠題材故事或市場情緒補分。
8. 技術面只能作為重估確認或資金反應，不得單獨支撐價值重估高分。

## 七、AI 最終重估分數規則

請使用 0～100 分作為 AI 最終重估分數。

### 85～100：高度重估候選

通常需同時具備：

1. 舊標籤與新標籤轉換非常明確。
2. 有公司公告、法說會、財報、月營收、公司官網、產品資料或正式新聞支撐。
3. 新產品、新客戶、新應用或新產業趨勢已有實質證據。
4. 營收、毛利、EPS 或法人籌碼至少部分驗證。
5. 技術面或資金面已經開始反映。
6. 反證少，資料完整度高。

### 75～84：強重估觀察

通常具備：

1. 新標籤邏輯明確。
2. 有部分正式來源或主流媒體支撐。
3. 營收、財報、法人籌碼或技術面有初步改善。
4. 尚有部分資料缺口，例如營收占比、客戶名稱、毛利貢獻或出貨規模尚未完全確認。

### 65～74：中度重估觀察

通常具備：

1. 有重估線索，但硬證據不足。
2. 題材與公司有合理關聯，但仍偏推論。
3. 技術面或法人籌碼有轉強跡象。
4. 需要等待營收、法說會、財報或公告驗證。

### 50～64：低度重估

通常代表：

1. 有題材關聯，但證據薄弱。
2. 公司可能受惠，但營收、產品、客戶或財報支撐不足。
3. 技術或籌碼可能只是短線反彈。
4. 蹭題材風險偏高。

### 0～49：暫不構成價值重估

通常代表：

1. 沒有明確舊標籤到新標籤的轉換。
2. 只有新聞、社群或短線情緒。
3. 缺乏產品、營收、財報、公告或法人籌碼支撐。
4. 反證明顯，或資料嚴重不足。

## 八、價值重估判斷框架

請用以下五層證據判斷每檔股票。

### 第一層：標籤轉換

請回答：

1. 舊市場標籤是什麼？
2. 新市場標籤是什麼？
3. 新標籤是否比舊標籤具備更高估值想像？
4. 這個標籤轉換是已驗證、推論中，還是只是市場情緒？

### 第二層：產品與供應鏈證據

請回答：

1. 公司產品是否真的對應新題材？
2. 公司是在核心供應鏈、直接供應鏈、間接受惠，還是只是同產業沾邊？
3. 有沒有公司公告、法說會、年報、官網、主流產業新聞或法人摘要支撐？
4. 若只是推論，必須標示「推論型加分」。

### 第三層：營收與財報驗證

請回答：

1. 月營收是否改善？
2. 近 3 個月或近 6 個月營收趨勢是否轉強？
3. 毛利率、營益率、EPS 是否改善？
4. 新題材是否已反映在營收或財報？
5. 若尚未反映，必須標示「尚待驗證」。

### 第四層：法人、籌碼與技術確認

請回答：

1. 外資、投信、自營商或法人合計是否有轉買跡象？
2. 大戶或 TDCC 籌碼是否改善？
3. 融資融券是否有風險？
4. 技術面是否出現轉強訊號，例如站上 21MA、突破壓力、量能溫和放大、策略 A/B/C/D 訊號？
5. 技術面只能作為確認，不得單獨支撐價值重估高分。

### 第五層：反證與蹭題材風險

請回答：

1. 公司是否只是同產業沾邊？
2. 新聞是否只提產業，沒有提公司？
3. 題材是否只來自論壇、社群或短線市場情緒？
4. 公司是否有營收衰退、毛利惡化、法人賣超、融資過熱等反證？
5. 是否存在公司澄清、產品無關、營收占比極低等負面證據？

## 十、加分分類規則

每檔股票都必須區分以下四類。

### 1. 本地量化底稿

請列出系統提供的本地分數，例如：

- rerating_score
- verification_score
- local_rerating_composite_score
- radar_score
- technical_score
- revenue_score
- chip_score

若資料未提供，請寫「資料未提供」。

### 2. 已驗證加分

只能包含較高可信度資料，例如：

- 公司公告
- MOPS
- 法說會
- 財報
- 月營收
- 年報
- 公司官網
- 主流財經或產業媒體明確報導
- 法人報告摘要
- 已可觀察的營收、毛利、EPS、法人籌碼或技術面改善

### 3. 推論型加分

可包含：

- 產業趨勢可能帶動需求
- 供應鏈位置可能受惠
- 題材與產品存在合理關聯
- 技術面或籌碼面可能提前反映
- 同族群共振

但必須標示：

- 這是推論，不是已驗證事實。
- 尚待驗證的關鍵資料是什麼。

### 4. 情緒型參考

可包含：

- 社群討論
- 論壇熱度
- 短線新聞熱度
- 市場資金追逐
- 題材擴散

限制：

- 情緒型參考不得單獨支撐高分。
- 情緒型參考只能作為輔助。
- 若只有情緒型參考，AI 最終重估分不得過高。

1. 市場可能買單故事：市場可能因為什麼題材、趨勢、客戶、產品、族群輪動或新標籤而重新評價。
2. 爆發條件：接下來哪些價量、法人、營收、公告、法說、客戶、產品或產業事件可能點火。
3. 待驗證訊號：哪些資料仍不足，後續需要追蹤什麼。
4. 失敗條件：哪些反證會讓想像降溫，或讓候選股不應進入前段排序。
5. 分類標籤：可標示為強證據重估型、早期想像型、籌碼點火型、題材擴散型、補漲候選型、高想像高風險型、蹭題材風險型或純財報型。

想像力很強但證據不足的股票可以保留為早期觀察，不得直接用想像補成高分。
